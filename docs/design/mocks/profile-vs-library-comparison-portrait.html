Clips initiative · decision support · Profile vs Library

Where does editable raw material live?

Two positions designed at full strength on the same dataset. Option A — a calm presentational Profile + a separate dense Library workbench (Aaron). Option B — Profile is the library, with a preview-public toggle (Ben). The honest contrast is the deliverable; neither is under-designed.

ADedicated Library page · Aaron

Two destinations, each doing one job

The public Profile stays purely presentational — a portfolio of curated published work, zero management chrome. The Library is an honest workbench: all recordings + clips, drafts + anon, filters, sort, edit entry, allowed to be dense because it's a tool.
