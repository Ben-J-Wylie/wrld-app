USAGE NOTES

How these compose

Every wizard screen has the same shell: back button · progress indicator · headline · supporting copy · primary content · primary CTA · skip (when allowed). Skip lives directly under the primary CTA — never above it, never in the header. Errors render inline beneath the offending input and color the input border live red. Loading states render in the input affordance (right side), not as a screen blocker. Permission pre-prompts always come before the OS prompt; if the OS prompt is denied, the wizard advances and the post-denial state lives downstream of onboarding (e.g., a creator with no location permission sees a banner on Go Live).
